Lab #1



This lab will give you practice with:

  • writing a scanner
  • using a map
  • using a Makefile

Thanks to this lab, you will figure out how tedious it is to write a scanner from scratch and you'll be happy in the next labs to use JLex, a Java scanner generator.

Objectives

Given an English text in form of a text file, you are to design a program to compute the list of all the words appearing in the text. For each word, you are to display a list of pairs (called information). A pair (information) is made of two integers, a line number (the line where the word appears) and a character position (the position of the first character of the word on the line). For example, given the following text file (all text are excerpts from THE PICTURE OF DORIAN GRAY by Oscar Wilde):

"I don't think I shall send it anywhere," he answered, tossing his
head back in that odd way that used to make his friends laugh at him
at Oxford. "No; I won't send it anywhere."

you program should output the following information:

  answered (1): [1:46]
  anywhere (2): [1:32] [3:33]
        at (2): [2:63] [3:1]
      back (1): [2:6]
     don't (1): [1:4]
   friends (1): [2:49]
        he (1): [1:43]
      head (1): [2:1]
       him (1): [2:66]
       his (2): [1:64] [2:45]
         i (3): [1:2] [1:16] [3:17]
        in (1): [2:11]
        it (2): [1:29] [3:30]
     laugh (1): [2:57]
      make (1): [2:40]
        no (1): [3:13]
       odd (1): [2:19]
    oxford (1): [3:4]
      send (2): [1:24] [3:25]
     shall (1): [1:18]
      that (2): [2:14] [2:27]
     think (1): [1:10]
        to (1): [2:37]
   tossing (1): [1:56]
      used (1): [2:32]
       way (1): [2:23]
     won't (1): [3:19] 

For example, the second line of output:

    anywhere (2): [1:32] [3:33]

tells us that the word anywhere appears two times in the text (see the "(2)"), at line 1 position 32, and at line 3 position 33.  All the words have been turned into lower case. Notice that all non-letter characters have been ignored except the character quote ( ' ) when it's part of a word like in don't or won't. For example, given the following input:

Don't you ever do 'THAT' again else you'll be
listening to me saying 'you're fired!!!'

leads to the following output:

The scanner you'll design must ignore any non-letter character (except the quote inside a word) including space, tab and newline characters. For example, the following input:

Getting started

In order to set up your environment for this lab and for all the comings labs you are first to follow those steps:

  • move into your home directory (in the terminal, this can be achieved by issuing the command cd)
  • create a new folder named compiler (in the terminal, this can be achieved by issuing the command mkdir compiler)
  • move into that folder (in the terminal, this can be achieved by issuing the command cd compiler)

Everything you'll do from now on should be located inside that folder. As for lab #1, perform the following steps:

  • download the archive lab1.tar.gz (see the supporting files section below)
  • move this archive inside the ~/compiler folder (in the terminal, this can be achieved by issuing the command  mv ~/Downloads/lab1.tar.gz ~/compiler)
  • move into the ~/compiler folder (in the terminal, this can be achieved by issuing the command cd compiler)
  • extract the archive there (in the terminal, this can be achieved by issuing the command tar xvf lab1.tar.gz)

What to do

You should complete the provided class skeletons to be able to run the provided Main class (do not change this class except the path of the input file). In addition you should design a Makefile , again by completing the provided Makefile skeleton. For the WordCount class, you are to use a Map to pair strings with a list of information. You can experiment two flavors of Map, the TreeMap and the HashMap (you will check the difference between those two classes).

What to turn in

Turn in all your Java sources (the .java files) files ONLY plus the Makefile.  Don't turn in any  .class  or  .txt  file!

To turn in your files, you MUST STRICTLY FOLLOW those steps:
  1. create a folder name after your name (please use the English version of your name, like  MarcGaetano)
  2. copy the files to turn in into that folder
  3. zip that folder: in the terminal, go on the parent folder of your folder and issue the command:  zip -r YourFolder.zip YourFolder/ . This will produce the archive file YourFolder.zip
  4. upload the archive YourFolder.zip on the eLearning platform

Labs turned in not following those rules won't be graded!

Supporting files

The supporting files are the Java classes skeleton you are to complete, plus a number of example text files with the corresponding expected output (outputN.txt is the expected output for textN.txt input file). All the supporting files are grabbed into the archive lab1.tar.gz you need to download and extract inside the ~/compiler folder.
