Specifications

For this assignment you will implement five Java classes: Type, SymInfo, SymTable, DuplicateSymException, and EmptySymTableException.  The Type class is used to define the possible types of a SymInfo object.  The SymTable class will be used by the compiler to represent a symbol table, a data structure that stores the identifiers declared in the program being compiled (e.g., function and variable names) and information about each of them (e.g., its type, where it will be stored at runtime). The symbol table will be implemented as a List of HashMaps. Eventually, each HashMap will store the identifiers declared in one scope in the program being compiled.

The HashMap keys are Strings (the declared identifier names) and the associated information are SymInfo objects. For now, the only information in a SymInfo will be the type of the identifier, represented using a Type object. Later on, you will enrich the SymInfo class to store more information.

The DuplicateSymException and EmptySymTableException classes define exceptions that can be thrown by methods of the SymTable class.

The Type class

You must implement a Type class such that you can identify the following types: Error type, Integer type, Bool type, Void type, String type, Function type, Struct type, Struct Definition type. A Type object doesn't have any attribute except the Struct Type which stores the name of the corresponding struct in the form of an IdNode (see ast.java). Given a Type object, we only need to know what kind of type it is. To achieve that, you should manage to define height isXXX functions: isErrorType, isIntType, isBoolType, isVoidType, isStringType, isFnType, isStructType, isStructDefType. Additionnally, you should override the methods equals and toString. You are free to implement the Type class the way you like.

The SymInfo class

You must implement the following SymInfo constructor and public methods (and no other public or protected methods):

  • SymInfo(InfoType type) : this is the constructor; it should initialize the SymInfo to have the given type.
  • Type getType()  : return this SymInfo's type.
  • String toString() :  return this SymInfo's type as a String (this method will be changed later in a future lab when more information is stored in a SymInfo).

The SymTable class

The SymTable must be implemented using a List of HashMaps. (think about the operations that will be done on a SymTable ) The HashMaps must map a String to a SymInfo. This means that the SymTable class will have a (private) field of type List<HashMap<String,SymInfo>>.

You must implement the following SymTable constructor and public methods (and no other public or protected methods):

  • SymTable() :   this is the constructor; it should initialize the SymTable's List field to contain a single, empty HashMap.
  • void addDecl(String name, SymInfo sym) throws DuplicateSymException, EmptySymTableException  :  if this SymTable's list is empty, throw an EmptySymTableException. If either name or sym (or both) is null, throw a NullPointerException. If the first HashMap in the list already contains the given name as a key, throw a DuplicateSymException. Otherwise, add the given name and sym to the first HashMap in the list.
  • void addScope() :  add a new, empty HashMap to the front of the list.
  • SymInfo lookupLocal(String name) :   if this SymTable's list is empty, throw an EmptySymTableException. Otherwise, if the first HashMap in the list contains name as a key, return the associated SymInfo; otherwise, return null.
  • SymInfo lookupGlobal(String name)  :  if this SymTable's list is empty, throw an EmptySymTableException. If any HashMap in the list contains name as a key, return the first associated SymInfo (i.e., the one from the HashMap that is closest to the front of the list); otherwise, return null.
  • void removeScope() throws EmptySymTableException :   if this SymTable's list is empty, throw an EmptySymTableException; otherwise, remove the HashMap from the front of the list. To clarify, throw an exception only if before attempting to remove, the list is empty (i.e. there are no HashMaps to remove).
  • void print()  :  this method is for debugging. First, print “\nSym Table\n”. Then, for each HashMap in the list, print it, print a newline. Finally, print one more newline. All output should go to System.out.

The DuplicateSymException and EmptySymTableException classes

These two classes will simply define the two checked exceptions that can be thrown by the SymTable class. Each exception must be able to be created using a constructor that takes no arguments.

The IdNode class

The IdNode class is defined in the ast.java file. You must change this class and add the following methods:

  • public void link(SymInfo info) : link the given SymInfo to the IdNode (i.e. store this SymInfo as attribute)
  • public SymInfo info() : return the SymInfo linked to this IdNode
  • public String name() : return the name (a String) of this IdNode

Those changes will be useful for the next labs.
